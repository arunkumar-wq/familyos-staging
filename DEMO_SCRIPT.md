**Duration:** 3–5 minutes
**Audience:** Investors, potential users, partners
**Tone:** Confident, clear, conversational

---

**[Slide or blank screen before login]**

## Demo Checklist (Presenter Notes)

Before the demo, ensure:

- [ ] App is running at staging URL or localhost:3000
- [ ] Database is freshly seeded (`npm run seed`)
- [ ] Browser is logged out (shows login page)
- [ ] Browser window is full-screen, no bookmarks bar
- [ ] A test document ready to upload (passport image or PDF works best)

## Backup Talking Points (if Q&A comes up)

**"Is the data secure?"**
> "Yes. JWT authentication, bcrypt password hashing, role-based access control on every API call, and multi-tenancy — every query is scoped by family. No family can see another family's data."

**"Does it work on mobile?"**
> "Fully responsive. Optimized for iPhone and iPad. The document upload even supports camera capture on mobile devices."

**"What if I have more than 5 family members?"**
> "The pro plan supports up to 10 members. We're building enterprise plans for larger families and multi-generational setups."

**"How does the AI work?"**
> "Currently we use Tesseract OCR for document reading with a rules-based analysis engine. Our roadmap includes GPT-4 Vision and Claude integration for even deeper document understanding."

**"What about document storage?"**
> "Files are securely stored with UUID filenames — no one can guess a file path. For production, we're migrating to encrypted cloud storage with signed URLs."

**"Can family members upload their own documents?"**
> "Absolutely. Based on their permission level, members can upload to their own vault, view shared documents, or have full admin access."
